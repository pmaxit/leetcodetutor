## How It Works

### Student Conversation Flow

```
1️⃣  INITIALIZATION
    Student: "Start interview for Two Sum"
    ↓
    Backend: /api/interviewer/init
    ↓
    Response: "### Initial Guidance\n\nWhat are the two boundary 
              conditions you need to check? Start at opposite ends..."
    ↓
    Session: currentHintIndex = 1 (Hint 1 shown)

2️⃣  ENGAGEMENT
    Student: "I'm stuck"
    ↓
    Backend: /api/chat
    ↓
    LLM checks: currentHintIndex=1, hints available, not asking solution
    ↓
    Response: "### Second Hint\n\nEach pointer moves toward the 
              optimal solution. When does your left pointer advance..."
    ↓
    Session: currentHintIndex = 2

3️⃣  ADDITIONAL HELP
    Student: "Still confused"
    ↓
    Backend: /api/chat
    ↓
    LLM checks: currentHintIndex=2, hints available, not asking solution
    ↓
    Response: "### Final Hint\n\nWatch for off-by-one errors in the 
              termination condition..."
    ↓
    Session: currentHintIndex = 3

4️⃣  SOLUTION REQUEST
    Student: "Show me the solution"
    ↓
    Backend: /api/chat
    ↓
    LLM checks: currentHintIndex=3, hintsExhausted=true, asking solution
    ↓
    Response: Full reference solution code
    ↓
    Session: currentHintIndex stays at 3
```

### Key Decision Points

The AI interviewer makes these decisions:

```
┌─ User asks for solution?
│  ├─ NO → Use hints for Socratic nudges (don't show code)
│  └─ YES → Are hints exhausted?
│     ├─ NO → Suggest showing remaining hints first
│     └─ YES → Show full solution code ✓

┌─ Showing hint?
│  └─ YES → Increment currentHintIndex for next request
│     └─ NO → Keep index same
```

✓ Each hint is 15-20 words (concise, not verbose)
✓ Hint 1 focuses on state/problem framing
✓ Hint 2 focuses on algorithm/recurrence
✓ Hint 3 focuses on edge cases/optimization
✓ No working code in hints (questions only)
✓ Consistent pattern across problem types
✓ No hint overlaps or redundancy
