 QUICKEST START (3 seconds)
═══════════════════════════════════════════════════════════════════════════════

$ ./deploy.sh

That's it. Wait for output showing:
  ✅ Public URL: https://abc123.ngrok-free.app
  ✅ All services running
  ✅ Health checks passed


═══════════════════════════════════════════════════════════════════════════════
 WHAT DEPLOY.SH DOES
═══════════════════════════════════════════════════════════════════════════════

1. Verifies prerequisites (Node.js, npm, ngrok, LM Studio)
2. Checks ports are available
3. Installs server dependencies (npm install)
4. Installs client dependencies (npm install in client/)
5. Verifies .env configuration
6. Tests server health
7. Starts Node.js server (port 3005)
8. (Optional) Starts React dev server (port 5173)
9. Starts ngrok tunnel (creates public HTTPS URL)
10. Displays all access points
11. Runs health checks
12. Stays running (Ctrl+C to stop)


═══════════════════════════════════════════════════════════════════════════════
 STOP EVERYTHING
═══════════════════════════════════════════════════════════════════════════════

$ ./stop-deployment.sh

Cleanly stops:
  • Node.js server
  • React client (if running)
  • ngrok tunnel
  • All subprocesses

Takes ~3 seconds


═══════════════════════════════════════════════════════════════════════════════
 CUSTOMIZATION
═══════════════════════════════════════════════════════════════════════════════

Change server port:
  $ SERVER_PORT=8080 ./deploy.sh

Change client port:
  $ CLIENT_PORT=3000 ./deploy.sh

Both:
  $ SERVER_PORT=8080 CLIENT_PORT=3000 ./deploy.sh

No client (server only):
  $ ./deploy.sh
  Then answer 'n' to "Start React dev server?"

═══════════════════════════════════════════════════════════════════════════════
 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

If deploy.sh fails immediately:

1. Check ngrok is authenticated:
   $ ngrok config check

2. Verify ngrok is installed:
   $ ngrok --version

3. Check Node.js:
   $ node --version

4. Check npm:
   $ npm --version

5. Check LM Studio is running:
   $ lms status
   (If offline, run: lms and start Local Server)

6. Check port 3005 is free:
   $ lsof -i :3005
   If used, either close that app or run:
   $ SERVER_PORT=8080 ./deploy.sh


If server won't start:

$ tail -50 /tmp/interview-platform/server.log

Common issues:
  • Dependencies missing: npm install
  • Port in use: use different port
  • .env missing: create with required fields
  • LM Studio offline: start LM Studio


If ngrok tunnel fails:

1. Check internet connection:
   $ ping 8.8.8.8

2. Check ngrok not already running:
   $ pkill -f ngrok
   Then retry: ./deploy.sh

3. Check ngrok auth:
   $ ngrok config add-authtoken <YOUR_TOKEN>

4. Check ngrok status:
   $ curl http://127.0.0.1:4040 (if tunnel running)

 USEFUL COMMANDS
═══════════════════════════════════════════════════════════════════════════════

Deploy:
  $ ./deploy.sh

Deploy without client:
  $ ./deploy.sh
  # Answer 'n' to React dev server prompt

Deploy with custom port:
  $ SERVER_PORT=8080 ./deploy.sh

Deploy in background:
  $ nohup ./deploy.sh > /tmp/deploy.log 2>&1 &

Stop everything:
  $ ./stop-deployment.sh

Check server health:
  $ curl http://localhost:3005/api/health | jq .

Check ngrok status:
  $ curl http://127.0.0.1:4040/api/tunnels | jq .

View ngrok requests:
  $ open http://127.0.0.1:4040

Kill specific service:
  $ kill <PID>      # Graceful
  $ kill -9 <PID>   # Force

Check port usage:
  $ lsof -i :3005

Get processes:
  $ ps aux | grep -E "(npm|ngrok)" | grep -v grep
